# Engine Wrapper Documentation

## Overview

The `EngineWrapper` provides a convenient interface to the existing engine physics pipeline (`PintleEngineRunner`) for use in the robust DDP controller. It estimates engine performance from feed pressures with caching for efficient DDP rollouts.

## Usage

### Basic Usage

```python
from engine.control.robust_ddp import EngineWrapper, EngineEstimate
from engine.pipeline.io import load_config

# Load engine configuration
config = load_config("configs/default.yaml")

# Create wrapper
wrapper = EngineWrapper(config, cache_size=128)

# Estimate from feed pressures
P_d_F = 3e6  # Fuel feed pressure [Pa]
P_d_O = 3.5e6  # Oxidizer feed pressure [Pa]

estimate = wrapper.estimate_from_pressures(P_d_F, P_d_O)

# Access results
print(f"Chamber pressure: {estimate.P_ch/1e6:.2f} MPa")
print(f"Thrust: {estimate.F/1000:.2f} kN")
print(f"Mass flows: {estimate.mdot_F:.3f}, {estimate.mdot_O:.3f} kg/s")
print(f"Mixture ratio: {estimate.MR:.2f}")
print(f"Injector pressure drops: {estimate.injector_dp_F/1e6:.2f}, {estimate.injector_dp_O/1e6:.2f} MPa")
```

### Convenience Function

```python
from engine.control.robust_ddp import estimate_from_pressures

# One-off estimation (creates temporary wrapper)
estimate = estimate_from_pressures(P_d_F, P_d_O, config)
```

### Caching

The wrapper includes LRU caching to avoid redundant evaluations during DDP rollouts:

```python
wrapper = EngineWrapper(config, cache_size=128)

# First call - evaluates
estimate1 = wrapper.estimate_from_pressures(3e6, 3.5e6)

# Second call with same pressures - uses cache
estimate2 = wrapper.estimate_from_pressures(3e6, 3.5e6)  # Fast!

# Check cache stats
stats = wrapper.get_cache_stats()
print(f"Cache size: {stats['size']}/{stats['max_size']}")
print(f"Usage: {stats['usage']*100:.1f}%")

# Clear cache
wrapper.clear_cache()
```

## EngineEstimate Structure

The `EngineEstimate` dataclass contains:

```python
@dataclass
class EngineEstimate:
    P_ch: float              # Chamber pressure [Pa]
    F: float                 # Thrust [N]
    mdot_F: float            # Fuel mass flow [kg/s]
    mdot_O: float            # Oxidizer mass flow [kg/s]
    MR: float                # Mixture ratio (O/F)
    injector_dp_F: float     # Injector pressure drop (P_d_F - P_ch) [Pa]
    injector_dp_O: float     # Injector pressure drop (P_d_O - P_ch) [Pa]
    stability_metrics: Optional[Dict[str, Any]]  # Stability analysis results
    diagnostics: Optional[Dict[str, Any]]        # Full diagnostics dict
```

## Stability Metrics

The wrapper extracts stability metrics from the pipeline results if available:

**Full Stability Analysis** (when available):
```python
{
    "stability_state": "stable" | "marginal" | "unstable",
    "stability_score": float,  # 0-1 score
    "chugging": {...},         # Chugging analysis
    "acoustic": {...},         # Acoustic modes
    "feed_system": {...},      # Feed system stability
}
```

**Minimal Injector Stiffness** (fallback):
```python
{
    "stability_state": "unknown",
    "stability_score": np.nan,
    "injector_dp_frac_F": float,  # P_d_F - P_ch / P_d_F
    "injector_dp_frac_O": float,  # P_d_O - P_ch / P_d_O
    "injector_stiffness_ok": bool,  # True if both > 0.1
}
```

## Implementation Details

### Feed Pressure Approximation

The wrapper uses feed pressures (`P_d_F`, `P_d_O`) as approximations for tank pressures when calling the pipeline:

```python
results = runner.evaluate(
    P_tank_O=P_d_O,  # Approximate tank pressure
    P_tank_F=P_d_F,  # Approximate tank pressure
    silent=True,
)
```

**Rationale:**
- The dynamics model tracks `P_d_F` and `P_d_O` as states
- The pipeline expects `P_tank_O` and `P_tank_F` (upstream of feed losses)
- For controller use, we approximate `P_tank ≈ P_d` since the pipeline internally accounts for feed losses
- This is a reasonable approximation for control purposes

**Note:** For more accuracy, one could work backwards from `P_d` to `P_tank` using the feed loss model, but this adds complexity and the approximation is sufficient for control.

### Error Handling

If the pipeline evaluation fails (e.g., infeasible pressure combination), the wrapper returns an estimate with NaN values:

```python
estimate = wrapper.estimate_from_pressures(infeasible_P_F, infeasible_P_O)
# estimate.P_ch = np.nan
# estimate.F = np.nan
# estimate.diagnostics["error"] = "error message"
```

This allows the DDP controller to handle infeasible states gracefully.

### Cache Key Generation

Cache keys are generated by:
1. Rounding pressures to 3 decimal places (reduces cache misses from floating point precision)
2. Creating MD5 hash of `"{P_d_F:.3f}_{P_d_O:.3f}"`

This ensures that similar pressures (within 1 Pa) map to the same cache entry.

## Integration with Dynamics

The wrapper is designed to be used with the dynamics model:

```python
from engine.control.robust_ddp import step, EngineWrapper

# In DDP rollout:
x = np.array([P_copv, P_reg, P_u_F, P_u_O, P_d_F, P_d_O, V_u_F, V_u_O])
u = np.array([u_F, u_O])

# Get mass flows from engine physics
estimate = wrapper.estimate_from_pressures(x[IDX_P_D_F], x[IDX_P_D_O])
mdot_F = estimate.mdot_F
mdot_O = estimate.mdot_O

# Step dynamics
x_next = step(x, u, dt, params, mdot_F, mdot_O)
```

## Performance Considerations

1. **Caching**: Essential for DDP rollouts which evaluate many similar pressure combinations
2. **Silent Mode**: Wrapper uses `silent=True` to suppress logging during rollouts
3. **Cache Size**: Default 128 entries is sufficient for most DDP horizons (N=50)
4. **Cache Key Precision**: 3 decimal places balances cache hit rate vs. accuracy

## Limitations

1. **Feed Pressure Approximation**: Uses `P_d ≈ P_tank` which is approximate
2. **No Feed Loss Backward**: Doesn't work backwards from `P_d` to `P_tank` using feed loss model
3. **Error Handling**: Returns NaN on failure - DDP must handle this

## Future Improvements

1. **Backward Feed Loss**: Compute `P_tank` from `P_d` using feed loss model for accuracy
2. **Adaptive Cache**: Adjust cache size based on DDP horizon
3. **Parallel Evaluation**: Support parallel evaluation for batch DDP rollouts
4. **Gradient Estimation**: Provide gradients `∂F/∂P_d` for DDP
